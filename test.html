Remote communications - Zoom and Webex

Existing affordances, visual variables and interaction flows in case of delivering an academic presentation.

Affordances

  1. Audio controls: a user can mute or unmute the microphone, select input microphone or the output speakers. Also you can mute/unmute other participants.
  2. Video controls: a user can turn on or off his camera, select which camera to use if multiple are present, use virtual backgrounds / blur, change orientation.
  3. Button for the participants list.
  4. Sharing the screen: the user can share the screen with other participants, which is an important affordance for an academic presentation.
  5. Having a reaction: a user can present a reaction -emoji- so he can interact with other users without desturbing the proffesor, for example.
  6. Apps: integration with third party apps so the workflow is facilitated.
  7. Whiteboards: a user can start a whiteboard, which is also important in academic presentation.
  8. Chat: it provides the users with the posibility of communicating non-verbaly, writing in chat. For example having a question in chat so that the presenter will not be interupted.

Visual variables

  1. Screen layout: the posibility to have the user speaking or presenting "in front", in a big picture, while the other users are listed in smaller pictures. Another possibility is to have all the users be in the gallery mode, having equal proportions.
  2. Video quality: in very large meetings, a big chunk of data has to be rendered, so in this cases having the possibility to adjust the video quality of every participant is important.
  3. The possibility to change the displayed name.
  4. The possibility to share specific applications open at that time, screens, portions of the screen or whiteboards.

Interaction flow

  1. Before-presentation: the presenter can set up the meeting details, share the meeting link, test the audio and video settings, like muting everybody on join.

Comparing the visual metaphors and idioms and suggesting new ones.

The image above is showing the bar of actions for Webex(left) and Zoom(right). The visual metaphors and idioms are fairly similar, like the mute/unmute and start/stop video butons. The same is for the share and record buttons. Webex has a quick button for raising a hand, while in Zoom all the reactions are placed in the Reactions menu (so one click to raise a hand in Webex vs. 2 clicks in Zoom). Webex has a "More options" idiom, with the visual metaphor of 3 dots. For the "Exit the meeting" button Webex has just a visual metaphor, meanwhile Zoom has a red button with the idiom "End". The same if for the "Apps", "Participants" and "Chat" options, Webex having just the visual metaphros, while Zoom has both idioms and visual methaphors. Zoom has a "Whiteboard" option in the tools bar, while Webex has it stored in the "Share" button. All the visual metaphors and idioms are fairly similar.

For the text-only interaction part, supposing that every interaction possible with the app would be possible in text-only (for example if the user wants to open the gallery of participants a new window will appear), all the commands would be:

  • mic_mute / mic_unnute : microphone controls
  • get_mic_inputs : prints the input devices for audio
  • set_mic_input *the device* : sets the audio input device
  • get_speaker_outputs : prints the output devices for audio
  • set_speaker_output *the device* : sets the audio output device
  • start_video : starts the camera
  • stop_video : stops the camera
  • get_video_devices : prints the video devices
  • set_video_device *the device* : sets the device used for video
  • print_participants : prints the participants of the meeting
  • to_chat : opens a new window where the chat is displayed
  • exit : exits the meeting
  • view_speaker : opens a window which visualises the video of the speaker
  • view_gallery : opens a window which visualises all the participant's videos

Discussing the cognitive overload in special contexts

The apps are quite easy to understand and work with, with 3 colors, pretty good idioms and visual methaphors, the menus are not having too much options and are not cascading. So, it can be said that Webex and Zoom are pretty intuitive apps.

Some special contexts where cognitive overload could appear would be when every participant has a camera on, microphone on and is sharing some content. Also, the chat is heavilly used and a normal user can't see all the messages. So it can get quite annoying for a user. Both Webex and Zoom have meeting settings that can prevent most of this previously said occurences. For example, the owner of the meeting can choose if a user can share some content at all or if multiple participants can share a screen at the same time.
